### 선형독립과 Rank

벡터 a,b,c로 정의된 행렬이 있다고 가정해보자

[1 0 -1]

[2 0 1]

[0 1 0]

[a b c]


A∈R** 3 * 3 (행렬 A는 3 by 3 행렬에 속한다)

이제, 이 행렬이 선형독립을 만족하는지 즉, 3차원 공간을 span할 수 있는지 살표보자.

이를 위해서는 벡터 a와 b, a와 c, b와 c를 비교해야 한다.

![필기1](img/선형대수학2_1.jpg)


**따라서 A∈R ** 3 * 3는 선형독립을 만족하며 Rank는 3이다.** 


### Rank가 떨어지는 경우 (종속)

만약 행렬 A가 아래와 같다면

[1 -1 0]

[2 -2 0]

[0 0 1]

[a b c]

열의 기준으로 보았을 때, 벡터 b = - a 이다.

이처럼 1열과 2열은 서로 종속이다. 3열은 앞의 두 열로 만들 수 없어서(두 열의 상수 곱으로 만들 수 없어서) 독립이다. 그러므로 Rank=2이다. 



열의 기준으로 보아도 r2​=2r1​. 즉, 두번쨰 행이 첫 번째 행의 상수배이다. 그러므로 선형독립이 될 수 없다. (다시말해 적어도 한 행은 새로운 정보를 추가하지 못한다)

*Rank는 서로 선형독립인 행(또는 열)의 최대 개수이다.*

이처럼 종속인 경우 Rank가 떨어진다.

### Null Space란 무엇인가?

전체 n차원 공간이 있는데, 여기서 현재 벡터 모음으로 만들지 못하는 부분이 존재한다.

 우리는 그것을 **Null space**라고 부른다. 
 
 Null space가 채워지는 순간(즉, 행렬에 벡터가 곱해지는 순간) 해당 space가 완전히 채워지게 된다.  

 
 이에 대해 구체적으로 알아보자.

 ![필기4](img/선형대수학2_4.jpg)

 위의 필기를 보면, 행렬A의 벡터 a와 b는 상수의 곱으로 나타낼 수 있기 때문에 종속이며 rank=1이다. 그러므로 2차원 공간을 span하지 못한다. 

 그런데, 벡터 x를 곱해주니 0을 만들 수 있다. 

 **그러므로 행렬과 벡터의 곱이 의미하는 것은 원래의 vector space가 만들지 못했던 차원의 부분을 x vector가 만드는 것이다.**

 이때 기존의 vector space가 만든 부분을 Range space라고 부르며, 만들지 못한 부분 그래서 x vector가 만든 부분을 Null space라고 부른다. 

 위에서 언급한, n차원의 공간에서 현재 벡터 모음으로 만들지 못하는 부분이 Null space라는 정의가 이것으로 설명된다. 

*물론, 이 설명은 수학적으로 엄밀하지 않은 것 같다. GPT에 설명에 의하면, '행렬 A의 열들이 선형종속이면 출력 공간(Range space)은 낮은 차원의 부분공간이 되며,그 결과 입력 공간에는 곱했을 때 0으로 사라지는 방향들이 생기는데, 이 입력 공간의 부분공간을 Null space'라고 한다.*

암튼, 중요한 것은

**선형종속인 행렬 A에 어떤 벡터x를 곱했더니 Ax=0이 된다면,**

**그 x는 행렬 A가 구분하지 못하는 입력 방향, 즉, Null space에 속한 벡터라는 뜻이다!!**

*Rank가 줄어드면 그만큼 Null space 차원이 반드시 생긴다. 즉, 선형 종속 -> Null space 손실 -> 정보손실*

- Null space가 있다? 선형종속이다.
- 선형종속이다? rank < 열의 개수
- 결국 행렬은 입력 정보를 완전히 보존하지 못한다. 

![필기5](img/선형대수학2_5.jpg)

위 사진에서 빠져도 되는 vector가 있다. 종속인 벡터 둘 중 하나가 빠져도, 남은 두 벡터로 2차원 공간을 만들 수 있다. 이 두 vector가 각각 Basis vector가 되는 것이다.

*Basis Vector는 n차원 공간을 span할 수 있는 각각의 vector임을 떠올리자*


### Matrix as Linear Operator
#### 행렬 자체가 선형연산자이다?!

행렬 A와 벡터 x를 곱해서 벡터 b가 나왔다. 

(위에서 행렬과 벡터를 곱하면 벡터가 나옴을 언급했다)

이를 통해 알 수 있는 것은 행렬 A가 벡터 x를 벡터 b로 바꿔주었다. 

**행렬에 벡터를 곱한다는 것의 의미**

Vector to vector: 왜 행렬에 벡터를 곱했더니 벡터가 이동했느냐? 그것은 행렬에 의해서 좌표공간이 뒤틀리는 즉, 변형되기 때문이다! 라고 해석할 수 있다. 

![필기6](img/선형대수학2_6.jpg)

위의 그림에서 행렬 A는 좌표공간을 비틀어서 새로운 공간을 만들고, 이 공간에 새로운 단위를 정의한다. 

이 비틀어진 a,b 공간에서 (1,1)을 찍어보면, 

**a와 b로 좌표 축을 세웠을 때(쉽게 말해 a와 b의 관점으로 좌표계를 보았을 때) (1,1)은 원래 우리가 보던 세계에서는 (3,3)으로 나타날 것이다.**

이런 방식으로 행렬을 설명할 수 있다!!
